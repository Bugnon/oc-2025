Planning

Le module aura lieu entre les vacances d'automne et de Noël. Un planning détaillé se trouve sur la page d'accueil de ce site.

Le projet

Vous allez créer un site web individuel. Le site doit avoir un certain nombre de caractéristiques qui doivent être documenté dans le cahier des charges qui doit être validé au début du cours et doit contenir une maquette du site.

Le projet sera évalué durant une présentation finale à la fin du module.

Cahier des charges

Le cahier des charge décrit à quoi le site doit ressembler en terme de contenu, style et organisation. Le cahier des charges doit être proposé par l'élève avant le 28 octobre 2025.

Le cahier des charge doit être documenté dans le fichier README.md à la base du du dossier du projet web. Il doit impérativement comprendre une maquette du site web.

Le cahier des charge comprend des éléments imposés :

  • Le site web doit contenir au minimum quatre pages dans quatre fichiers HTML différents, dont une page d’accueil.
  • La page d’accueil doit présenter le sujet traiter et offrir au visiteur une table des matières du site web avec un lien vers les pages correspondantes et une brève description du contenu.
  • Il doit y avoir une barre de navigation pour naviguer entre les quatre pages.
  • Chaque page doit contenir au moins 100 mots.
  • Le sujet traité doit être liés aux STIM et le contenu (texte, images, équations, code, animations, etc.) doit être cohérent, pertinent et maîtrisé par l’élève.
  • La page web doit contenir soit des équations, soit du code informatique, soit un formulaire dynamique.
  • Le site web doit être stylé avec une feuille CSS.
  • Il doit y avoir une animation réalisée avec canvas. (Exemples)
  • Le code doit être sur GitHub est être déployé sur le web avec GitHub Pages.
  • La page doit pouvoir s’adapter aux petits écrans (smartphone).

Voici les éléments du cahier des charge qui doivent être proposés par l'élève :

  • Le sujet traité et le contenu du site web ;
  • L'apparence (ou style) du site web ;
  • La structure du site web.

Thème du site web

Le thème est laissé au libre choix de l'élève, du moment qu'on peut intégrer les éléments imposés du cahier des charges de manière pertinente. On recommande un sujet sur lequel l'élève est à l'aise, par exemple :

  • Sujet du travail de maturité ;
  • Cours intéressant de gymnase ;
  • Autre intérêt lié à l'informatique, les sciences, mathématiques ou technologies ;
  • etc.

Maquette

Une maquette du site est un élément du cahier des charges et doit être intégré au fichier README.md qui le documente.

Une maquette est une représentation graphique du site web envisagé. Il doit donner une idée de l'apparence, de l'organisation du site. Il explicite aussi comment les éléments du cahier des charge vont être intégrés au site web.

La manière de créer la maquette est libre. La maquette peut être un diagramme réalisé avec papier et crayon et/ou collage, ou alors avec un logiciel (PowerPoint, outil graphique, Paint, etc.).

Présentation finale

La présentation finale certificative à lieu sur les trois ou quatre dernières périodes de la séquence. Le tirage de l’ordre de passage se fera par tirage au sort. Tous les élèves doivent être présents pour écouter les présentations de leurs pairs.

Pour chaque passage il y aura 7 minute de présentation et 5 minutes de questions.

  • Présentation (7 minutes)
    • Une présentation du cahier des charge et du thème choisi ;
    • Une présentation du site web construit;
    • Une exploration technique du code : HTML, CSS, animation canvas et autres éléments JS à l’aide de Chrome DevTools ;
    • Une démonstration de l’adaptation faite pour les mobiles ;
    • Une discussion des difficultés rencontrée et sur les solutions et adaptations choisies.
  • Questions (5 minutes)
    • Questions sur le sujet du site et l’implémentation ;
    • Questions de connaissance sur les technologies utilisée ;
    • Questions libres des autres élèves.

Critères d'évaluation

L'évaluation sera basée sur le site web produit et la présentation finale et la note se basera sur les critères suivants :

  • Respect du cahier des charges ;
  • Clarté de la présentation finale et des réponses aux questions ;
  • Pertinence, cohérence et maîtrise du contenu ;
  • Structure et esthétique du site ;
  • Maîtrise des technologies employées

Les outils

Nous utiliserons les outils suivants durant le module :

  • VS Code ainsi que l'extension HTML Preview pour la programmation.
  • Google Chrome et Chrome DevTools pour explorer et débugger des sites web.
  • GitHub, GitHub Desktop et GitHub Pages pour héberger le code, le partager et déployer le site web.
